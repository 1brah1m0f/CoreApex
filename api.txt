CoreApex — FastAPI Endpoint Specification
==========================================
Extracted from: frontend/src/api/index.ts, types/index.ts, mocks/index.ts,
                pages/CitizenPage.tsx, pages/inspector/Simulation.tsx,
                pages/executive/Analytics.tsx, pages/citizen/NewReport.tsx

Base URL  : /api/v1
Auth      : Authorization: Bearer <token>
Token src : localStorage["apexcore_token"]
On 401    : client clears token + redirects to /
Errors    : { "detail": "string" }  OR  { "message": "string" }
Note      : All list endpoints must return [] on empty, never null.

AUTH  (/auth)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

POST /auth/register                              ← PRIMARY (new)
  Body    : { name: str, email: str, password: str }
  Returns : { token: str, role: Role, user: { id: str, name: str, email: str } }
  Note    : Backend assigns role from user record (citizen by default, or preconfigured
            for staff). Returns JWT immediately — no separate login step needed.

POST /auth/login                                 ← PRIMARY (new)
  Body    : { email: str, password: str }
  Returns : { token: str, role: Role, user: { id: str, name: str, email: str } }
  Note    : Works for ALL roles (citizen, inspector, executive). Frontend redirects
            to the correct portal based on the returned role field.
            role → redirect:  citizen → /citizen,  inspector → /inspector,  executive → /executive

GET  /auth/me
  Auth    : required
  Returns : { id: str, name: str, role: Role, email: str, phone: str|None }

POST /auth/login/staff                           ← legacy, keep for compat
  Body    : { email: str, password: str }
  Returns : { token: str, role: "inspector"|"executive", user: { id, name, email } }

REPORTS  (/reports)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

POST /reports
  Auth    : required
  Body    : {
              title        : str,
              description  : str,
              address      : str,
              neighborhood : str | None,
              category     : str,             # "Yollar"|"Su kəməri"|"Elektrik"|"Abadlıq"|"Zibil"|"Digər"
              photo_url    : str | None,
              lat          : float | None,
              lng          : float | None
            }
  Returns : Report

POST /reports/ai-classify
  Auth    : required
  Body    : { photo_url: str }
  Returns : { category: str | None, title: str | None }
  Note    : Return 200 with nulls if AI cannot classify — do NOT return 422.

GET  /reports/mine
  Auth    : required (citizen)
  Query   : status?: ReportStatus
  Returns : list[Report]

GET  /reports
  Auth    : required (inspector, executive)
  Query   : status?, category?, neighborhood?, page?, limit?
  Returns : list[Report]

GET  /reports/{id}
  Auth    : required
  Returns : Report

PATCH /reports/{id}/status
  Auth    : required (inspector, executive)
  Body    : { status: ReportStatus }
  Returns : Report

POST /reports/{id}/sla-report
  Auth    : required (executive)
  Body    : {
              report_text        : str,
              recommended_action : str,
              severity           : str    # "low"|"medium"|"high"|"critical"
            }
  Returns : { message: str }

GET  /reports/export
  Auth    : query param ?token=<jwt>    ← window.open, no headers
  Query   : same filters as GET /reports
  Returns : CSV file (Content-Disposition: attachment)

IMPLEMENTATION NOTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Auth flow (frontend/src/pages/AuthPage.tsx):
   - Single page at /auth with Login / Register tabs.
   - POST /auth/login or POST /auth/register → { token, role, user }
   - Stores: localStorage["apexcore_token"] = token
             localStorage["apexcore_role"]  = role
             localStorage["apexcore_user"]  = JSON.stringify(user)
   - Redirects: citizen→/citizen  inspector→/inspector  executive→/executive
   - Role is entirely determined by the backend response — no role selector on frontend.

2. /reports/export passes token via ?token= query param because it uses
   window.open() — no Authorization header is possible.

3. Task.map_x = latitude, Task.map_y = longitude (frontend naming is inverted
   from convention — keep consistent with these field names).

4. Simulation /run and /risk-assess responses are displayed raw as JSON in a
   <pre> block. No schema enforced by frontend — return whatever shape is useful.

5. All auth endpoints return "token" (not "access_token") — frontend stores it
   in localStorage["apexcore_token"] and sends as Bearer.

6. category values used in frontend (Azerbaijani):
   "Yollar" | "Su kəməri" | "Elektrik" | "Abadlıq" | "Zibil" | "Digər"
   ai-classify must return one of these exact strings or null.

7. Proposal tag examples from mocks:
   "infrastruktur" | "ətraf mühit" | "nəqliyyat" | "təhsil" | "sosial" | "texnologiya"
